# V0.3.1 Batch Event Sourcing Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a stable China Athletics official event source that imports multiple backend candidates per run, reports a clear run summary, and keeps every candidate behind manual completion and review.

**Architecture:** Keep the existing `EventSource -> EventCandidate -> Event(draft)` boundary. Add a fixed, server-owned `chinaath_api` adapter instead of a generic arbitrary JSON API configuration, then refactor the runner so both the existing AI page extractor and the official API adapter feed the same batch persistence path. Structured official API fields are mapped deterministically; GLM/DeepSeek/OpenAI remain responsible only for unstructured page extraction.

**Tech Stack:** Node.js, TypeScript, Express, Prisma/PostgreSQL, Zod, Vitest, React 18, Ant Design 5, Vite, existing OpenAI-compatible provider adapter.

---

## Scope And Decisions

This plan starts from the code state inspected on 2026-07-10:

- `page_url` fetching, AI extraction, candidate editing, rejection, and acceptance as an `Event` draft already exist.
- GLM, DeepSeek, and OpenAI provider selection exists locally and its focused 13 tests plus full workspace typecheck pass.
- A page run currently returns exactly one candidate, even when the source is an event list.
- `search_query` and `rss` exist in the database enum but are not implemented by the runner.
- The China Athletics list endpoint was live-checked on 2026-07-10 and returned `success=true`, `totalCount=2830`, and `data.results` records.

Choose the fixed adapter approach:

1. **Recommended: fixed `chinaath_api` adapter.** The server owns endpoint, request body, response validation, batch limit, and field mapping. Admins cannot configure arbitrary request headers or POST bodies.
2. Generic `api_json` source. Rejected for this version because JSONPath mapping, credentials, arbitrary outbound requests, and per-site schemas create unnecessary SSRF and support risk.
3. AI-only parsing of the API response. Rejected because the source is already structured; deterministic mapping is cheaper, testable, and less likely to alter dates or names.

The public-product rules remain unchanged:

- Candidates never appear in the miniapp.
- Acceptance creates only `publishStatus=draft` records.
- Missing official registration links remain `null`; they must not be invented from the API host.
- Admins must manually add and verify `officialUrl` before accepting a candidate.
- Preserve `AI 整理，仅供参考，报名以官方为准。` and `前往官方确认`; do not introduce `立即报名`.

```typescript
import { z } from 'zod';
import { CHINAATH_PUBLIC_LIST_URL } from '../eventSourceConfig.js';
import type { SourceCandidateBatch } from './sourceCandidate.js';

const CHINAATH_LIST_ENDPOINT =
  'https://api-changzheng.chinaath.com/changzheng-content-center-api/api/homePage/official/searchCompetitionMls';
const DEFAULT_BATCH_SIZE = 20;

const raceSchema = z.object({
  raceId: z.union([z.number(), z.string()]),
  raceName: z.string().trim().min(1),
  raceGrade: z.string().nullable().optional(),
  raceTime: z.string().regex(/^\d{4}-\d{2}-\d{2}$/),
  raceAddress: z.string().nullable().optional(),
  raceItem: z.string().nullable().optional(),
  raceScale: z.union([z.string(), z.number()]).nullable().optional(),
});

const responseSchema = z.object({
  success: z.literal(true),
  data: z.object({
    results: z.array(raceSchema),
    totalCount: z.number().int().nonnegative(),
  }),
});

interface FetchChinaAthOptions {
  fetchImpl?: typeof fetch;
  pageNo?: number;
  pageSize?: number;
  cityHints?: string[];
}

export async function fetchChinaAthOfficialCandidates(
  options: FetchChinaAthOptions = {},
): Promise<SourceCandidateBatch> {
  const fetchImpl = options.fetchImpl ?? fetch;
  const pageNo = options.pageNo ?? 1;
  const pageSize = Math.min(Math.max(options.pageSize ?? DEFAULT_BATCH_SIZE, 1), 50);
  const response = await fetchImpl(CHINAATH_LIST_ENDPOINT, {
    method: 'POST',
    headers: {
      accept: 'application/json',
      'content-type': 'application/json',
      'user-agent': process.env.AI_INGEST_USER_AGENT || 'WorthRunBot/0.1',
    },
    body: JSON.stringify({
      provinceId: '',
      cityId: '',
      districtId: '',
      raceName: '',
      raceGrade: '',
      raceStartTime: '',
      pageNo,
      pageSize,
    }),
    signal: AbortSignal.timeout(15_000),
  });

  if (!response.ok) {
    throw new Error(`中国田协赛事接口请求失败：HTTP ${response.status}`);
  }

  const raw = await response.json();
  const parsed = responseSchema.safeParse(raw);
  if (!parsed.success) {
    throw new Error('中国田协赛事接口返回失败或结构不符合预期');
  }

  const hints = (options.cityHints ?? []).map(normalizeCityHint).filter(Boolean);
  const records = hints.length
    ? parsed.data.data.results.filter((race) => {
        const haystack = `${race.raceName} ${race.raceAddress || ''}`;
        return hints.some((hint) => haystack.includes(hint));
      })
    : parsed.data.data.results;

  return {
    totalAvailable: parsed.data.data.totalCount,
    candidates: records.map((race) => {
      const city = parseCity(race.raceAddress);
      const distances = parseRaceItems(race.raceItem);
      const quote = [race.raceName, race.raceTime, race.raceAddress, race.raceItem]
        .filter(Boolean)
        .join('；')
        .slice(0, 300);

      return {
        sourceExternalId: String(race.raceId),
        rawPayload: race,
        extractorVersion: 'chinaath-api-v1',
        aiModel: null,
        aiPromptVersion: null,
        candidate: {
          eventName: race.raceName,
          city,
          eventDate: race.raceTime,
          distanceItems: distances,
          signupStatus: 'unknown',
          signupDeadline: null,
          officialUrl: null,
          sourceName: '中国田协官方赛事目录',
          sourceUrl: CHINAATH_PUBLIC_LIST_URL,
          sourceLevel: 'official',
          runJudgement: 'unverified',
          judgementSummary: '中国田协赛事目录记录；报名状态和官方报名入口仍需人工确认。',
          judgementReasons: ['赛事名称、日期、地点和项目来自中国田协赛事目录'],
          suitableFor: [],
          notSuitableFor: [],
          tags: race.raceGrade ? [`田协${race.raceGrade}类`] : [],
          evidence: [{ field: 'sourceRecord', sourceUrl: CHINAATH_PUBLIC_LIST_URL, quote }],
          confidence: {
            eventName: 'verified',
            city: 'verified',
            eventDate: 'verified',
            distanceItems: distances.length ? 'verified' : 'pending_verify',
            officialUrl: 'pending_verify',
            signupStatus: 'pending_verify',
          },
        },
      };
    }),
  };
}

function parseCity(address?: string | null) {
  const parts = String(address || '')
    .split('/')
    .map((item) => item.trim())
    .filter(Boolean);
  return parts[1] || parts[0] || '待确认';
}

function parseRaceItems(value?: string | null) {
  if (!value) return [];
  try {
    const parsed = JSON.parse(value);
    return Array.isArray(parsed)
      ? parsed.filter((item): item is string => typeof item === 'string' && item.trim().length > 0)
      : [];
  } catch {
    return value.split(/[,，、]/).map((item) => item.trim()).filter(Boolean);
  }
}

function normalizeCityHint(value: string) {
  return value.trim().replace(/[市区县]$/, '');
}
```

**Step 3: Implement write decisions and persistence**

Create `apps/api/src/ai/persistEventCandidates.ts`:

```typescript
import { EventCandidateStatus, Prisma, prisma } from '@worth-running/database';
import type { SourceCandidate } from './sources/sourceCandidate.js';

type ExistingCandidate = { status: string } | null;

export function decideCandidateWrite(existing: ExistingCandidate, _duplicateEvent: boolean) {
  if (!existing) return 'create' as const;
  if (existing.status === 'new' || existing.status === 'needs_review') return 'update' as const;
  return 'skip_reviewed' as const;
}

export interface PersistSummary {
  fetched: number;
  created: number;
  updated: number;
  skippedReviewed: number;
  duplicateEvents: number;
  candidateIds: string[];
}

export async function persistEventCandidates(
  sourceId: string,
  items: SourceCandidate[],
): Promise<PersistSummary> {
  const summary: PersistSummary = {
    fetched: items.length,
    created: 0,
    updated: 0,
    skippedReviewed: 0,
    duplicateEvents: 0,
    candidateIds: [],
  };

  for (const item of items) {
    const candidate = item.candidate;
    const eventDate = candidate.eventDate
      ? new Date(`${candidate.eventDate}T00:00:00.000Z`)
      : null;
    const duplicate = eventDate
      ? await prisma.event.findFirst({
          where: { eventName: candidate.eventName, city: candidate.city, eventDate },
          select: { id: true },
        })
      : null;

    const existing = item.sourceExternalId
      ? await prisma.eventCandidate.findUnique({
          where: {
            sourceId_sourceExternalId: {
              sourceId,
              sourceExternalId: item.sourceExternalId,
            },
          },
        })
      : await prisma.eventCandidate.findFirst({
          where: {
            sourceId,
            eventName: candidate.eventName,
            city: candidate.city,
            eventDate,
          },
          orderBy: { createdAt: 'desc' },
        });

    const decision = decideCandidateWrite(existing, Boolean(duplicate));
    if (decision === 'skip_reviewed') {
      summary.skippedReviewed += 1;
      continue;
    }

    if (duplicate) summary.duplicateEvents += 1;
    const data = {
      sourceId,
      sourceExternalId: item.sourceExternalId,
      status: duplicate ? EventCandidateStatus.needs_review : EventCandidateStatus.new,
      eventName: candidate.eventName,
      city: candidate.city,
      eventDate,
      sourceUrl: candidate.sourceUrl,
      officialUrl: candidate.officialUrl,
      extractedData: candidate as Prisma.InputJsonObject,
      evidence: candidate.evidence as Prisma.InputJsonArray,
      confidence: candidate.confidence as Prisma.InputJsonObject,
      rawPayload: item.rawPayload as Prisma.InputJsonObject | null,
      extractorVersion: item.extractorVersion,
      duplicateEventId: duplicate?.id ?? null,
      aiModel: item.aiModel,
      aiPromptVersion: item.aiPromptVersion,
    };

    const saved = existing
      ? await prisma.eventCandidate.update({ where: { id: existing.id }, data })
      : await prisma.eventCandidate.create({ data });

    summary[existing ? 'updated' : 'created'] += 1;
    summary.candidateIds.push(saved.id);
  }

  return summary;
}
```

```typescript
export interface EventSourceRunSummary extends PersistSummary {
  sourceId: string;
  totalAvailable: number | null;
}

export async function runEventSource(sourceId: string): Promise<EventSourceRunSummary> {
  const source = await prisma.eventSource.findUnique({ where: { id: sourceId } });
  if (!source) throw new AiIngestError(404, '赛事源不存在');
  if (source.status !== 'active') throw new AiIngestError(400, '赛事源未启用');

  try {
    const batch =
      source.sourceType === 'page_url'
        ? await fetchPageCandidateBatch(source)
        : source.sourceType === 'chinaath_api'
          ? await fetchChinaAthOfficialCandidates({ cityHints: source.cityHints })
          : null;

    if (!batch) {
      throw new AiIngestError(400, '当前仅支持页面 URL 和中国田协官方接口赛事源');
    }

    const persisted = await persistEventCandidates(source.id, batch.candidates);
    const summary = {
      sourceId: source.id,
      totalAvailable: batch.totalAvailable,
      ...persisted,
    };

    await prisma.eventSource.update({
      where: { id: source.id },
      data: { lastRunAt: new Date(), lastRunStatus: formatRunStatus(summary) },
    });
    return summary;
  } catch (error) {
    // Preserve the current AiIngestError conversion and failed:lastRunStatus update.
    throw await recordSourceFailure(source.id, error);
  }
}

export function formatRunStatus(summary: Omit<EventSourceRunSummary, 'totalAvailable'>) {
  return [
    `success:fetched=${summary.fetched}`,
    `created=${summary.created}`,
    `updated=${summary.updated}`,
    `skipped=${summary.skippedReviewed}`,
    `duplicates=${summary.duplicateEvents}`,
  ].join(',');
}
```

## Release Acceptance Criteria

- An admin can create a China Athletics source without entering API URLs or headers.
- One manual run can return and persist up to 20 candidates.
- The run response and admin message show fetched, created, updated, skipped-reviewed, and duplicate counts.
- Re-running the same official source uses `sourceId + raceId` identity and does not create duplicate pending candidates.
- Accepted and rejected candidates are never overwritten by later source runs.
- Missing official registration links remain missing until a human supplies them.
- Candidate acceptance produces only an `Event` draft.
- Public miniapp APIs continue returning only published events.
- Existing page URL sources still use the configured GLM/DeepSeek/OpenAI provider and retain robots/domain protections.
- Full tests, typecheck, build, migration, and browser smoke test pass.

## Deferred Work

Keep these out of V0.3.1:

- Scheduled/cron runs.
- Generic JSON API configuration.
- Search-query and RSS execution.
- Automatic page traversal or detail-page crawling.
- Automatic official registration link discovery.
- Bulk acceptance, automatic publication, or miniapp changes.
- A dedicated source-run history table; operation logs and `lastRunStatus` are sufficient for this release.
