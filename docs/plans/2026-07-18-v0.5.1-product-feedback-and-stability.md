# WorthRun V0.5.1 产品反馈与线上稳定性 Implementation Plan

**Goal:** 在保留赛事纠错的基础上增加匿名产品反馈，并用请求追踪、5xx 小时聚合、健康概览和优雅停机增强单机 ECS 稳定性。

## Delivery

- 旧反馈默认归类为 `event_correction`；产品反馈使用 `product_feedback`，不关联赛事。
- 小程序从“我的”及关键错误状态进入原生产品反馈页，只提交文字、固定页面标识、版本和关联请求编号。
- 后台按赛事纠错与产品反馈分流，继续复用状态、权限、批处理和审计日志。
- API 错误返回服务端 UUID；5xx 只按小时、路由组和类别聚合，不保存正文、IP 或用户信息。
- 复用现有维护 cron 清理 30 天前错误指标，PM2 保持单进程，日志使用系统 logrotate。

## Verification

- 全量测试、类型检查、API/admin build 和小程序 TypeScript 检查通过。
- ECS 迁移前备份数据库；迁移后旧反馈数量和状态不变。
- 线上验证产品反馈 201、重复 200、非法内容 400、限流 429、健康接口和后台运行状态。
- API RSS 小于 220MB，维护任务峰值小于 120MB，不新增 cron、Redis、队列或常驻模块。
