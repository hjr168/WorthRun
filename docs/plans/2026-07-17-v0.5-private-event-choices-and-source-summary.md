# WorthRun V0.5.0 匿名选择与来源摘要 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 把现有“查看赛事 -> 收藏”推进为匿名跑赛选择和公开意向统计，同时让用户在小程序内查看可追溯的来源内容摘要。

**Architecture:** 使用轻量 `UserEventChoice` 保存匿名用户的“想跑 / 观望 / 已报名”，详情接口按赛事实时聚合三个公开数量，不保存独立计数器或参与者名单。来源摘要复用现有受 robots 和大小限制保护的页面抓取器及 GLM/DeepSeek/OpenAI 兼容层，生成可编辑草稿，经管理员确认后公开；小程序使用原生 WXML 页面读取公开摘要接口。正文仅在生成时临时使用，不保存完整网页，不采集截图。

**Tech Stack:** pnpm monorepo、TypeScript、Express、Prisma/PostgreSQL、React + Ant Design、微信小程序原生框架、Vitest、PM2、Nginx。

## Summary

V0.4.5 已完成 ECS 部署和生产验收。线上近 7 天已有 37 次详情访问、21 个详情用户、16 次官方入口复制、10 次收藏和 7 次分享，但偏好用户只有 1 个。V0.5.0 优先承接具体赛事上的用户行为，并增强赛事来源透明度。

本阶段包含：

1. 用户可以匿名标记“想跑 / 观望 / 已报名”，形成“我的选择”，所有用户可在赛事详情查看三个选项的聚合数量。
2. 管理员可以抓取赛事来源正文，调用当前配置的 AI 模型生成来源摘要草稿。
3. 摘要必须经过人工编辑、确认和发布，不能随 AI 生成自动公开。
4. 赛事详情增加“查看来源详情”，进入原生小程序来源摘要页。

本阶段明确不做赛事并排对比、网页截图、自动截图、参与者名单、订阅提醒或自动发布摘要。

验收目标：

- 用户可以设置、修改和清除匿名选择；赛事详情从第 1 票开始准确显示三个选项的公开数量。
- 切换选择时旧选项减少、新选项增加；清除后对应数量减少，聚合结果不产生计数漂移。
- “我的”页展示选择摘要，并优先展示已报名或想跑的下一场赛事。
- 来源摘要只基于实际抓取正文或已保存的结构化来源记录生成。
- 原生来源摘要页展示来源名称、来源标题、摘要、关键点、抓取时间、生成依据和原始来源入口。
- 无已发布摘要时，赛事详情不展示空入口。
- V0.4.5 发现来源变化时，已发布摘要标记为需要复核，但不自动覆盖。
- 不保存完整网页正文、截图、Cookie、IP、请求头或浏览器指纹。
- 保持单个 API PM2 进程，不新增 cron、Redis、消息队列或常驻浏览器。
- API RSS 不超过 220MB，单次摘要生成峰值不超过 180MB。

## Product Boundaries

- 用户自己的选择仅当前匿名身份可见；公开接口只返回三个聚合数量，不返回 `userKey`、参与者名单或用户画像。
- 公开数量定义为匿名用户意向统计，不代表官方报名人数；“已报名”仅代表用户自报。
- 收藏和匿名选择相互独立，不自动同步。
- 每个 `userKey` 对同一赛事只能保留一个选择；切换和删除后由数据库实时聚合，不维护易漂移的计数器。
- 当前匿名身份可被用户清除或重建，公开数量仅作为产品参考值，不对外宣称为精确去重人数。
- 来源摘要属于辅助理解内容，不替代原始来源和官方公告。
- AI 只能生成摘要草稿；`event_operator` 或 `content_reviewer` 人工确认后才能发布。
- 页面抓取继续遵守 `robots.txt`、允许域名、超时、正文大小和反爬识别规则。
- 抓取失败时不得让 AI 根据赛事名称自行补写摘要。
- 对结构化官方来源，可使用已保存的 `rawPayload/extractedData/evidence` 生成摘要，并明确标记“基于已保存来源记录”。
- `community/secondary/unknown` 来源不得成为公开来源摘要的唯一依据。
- 小程序来源摘要页不显示候选内部字段、置信度、管理员备注、原始响应或变更告警详情。
- 固定保留“AI 整理，仅供参考，报名以官方为准。”和“前往官方确认”。

## Data Model

### 匿名选择

```prisma
enum EventChoiceType {
  interested
  considering
  registered
}

model UserEventChoice {
  id        String          @id @default(cuid())
  userKey   String          @map("user_key")
  eventId   String          @map("event_id")
  choice    EventChoiceType
  createdAt DateTime        @default(now()) @map("created_at")
  updatedAt DateTime        @updatedAt @map("updated_at")

  event Event @relation(fields: [eventId], references: [id], onDelete: Cascade)

  @@unique([userKey, eventId])
  @@index([userKey, choice, updatedAt])
  @@index([eventId, choice])
  @@index([createdAt])
  @@map("user_event_choices")
}
```

### 来源摘要

```prisma
enum SourceSummaryStatus {
  draft
  published
  superseded
}

enum SourceSummaryBasis {
  page_text
  stored_source_record
}

model EventSourceSummary {

  sourceUrl     String             @map("source_url")
  sourceTitle   String?            @map("source_title")
  summary       String
  keyPoints     String[]           @map("key_points")
  limitations   String?
  contentHash   String             @map("content_hash")
  aiProvider    String             @map("ai_provider")
  aiModel       String             @map("ai_model")
  promptVersion String             @map("prompt_version")
  fetchedAt     DateTime           @map("fetched_at")
  generatedAt   DateTime           @default(now()) @map("generated_at")
  publishedAt   DateTime?          @map("published_at")
  staleAt       DateTime?          @map("stale_at")
  reviewedBy    String?            @map("reviewed_by")
  createdAt     DateTime           @default(now()) @map("created_at")
  updatedAt     DateTime           @updatedAt @map("updated_at")

  event Event @relation(fields: [eventId], references: [id], onDelete: Cascade)

  @@index([eventId, status, updatedAt])
  @@index([status, staleAt])

```

不新增截图、Blob、文件路径、完整正文或 HTML 字段。数据库只保存摘要结果、来源元数据和 SHA-256 内容哈希。

## Summary Generation Rules

### 输入来源优先级

1. 赛事 `sourceUrl` 可抓取且来源等级为 `official/trusted`：抓取页面正文，basis 为 `page_text`。
2. 页面是 PDF、结构化 API、被允许但不是 HTML，或当前无法重新抓取：读取已采纳候选保存的官方 `rawPayload/extractedData/evidence`，basis 为 `stored_source_record`。
3. 两种输入均不可用：生成失败，不创建草稿。

`officialUrl` 只在与赛事来源一致且后台已核验时作为备用抓取地址；不能抓取聚合报名页来冒充官方来源。

### 临时正文限制

- 复用 `fetchRobotsAllowedPage`。
- HTML 最大 400KB，抽取文本最大 30,000 字符，15 秒超时。
- 发送 AI 前再次压缩到最多 20,000 字符，保留标题、开头、日期/报名/地点关键词附近内容。
- 完成生成后立即释放正文，不写数据库和日志。
- 生成失败日志只记录来源域名、HTTP 状态和通用错误，不记录正文。

### AI 输出结构

```ts
{
  sourceTitle: string | null;       // 最多 120 字
  summary: string;                  // 80-400 字
  keyPoints: string[];              // 2-6 项，每项最多 120 字
  limitations: string | null;       // 最多 200 字
}
```

提示词要求：

- 只能使用给定来源内容。
- 不推测报名已开放、名额、路线、认证等级或截止时间。
- 来源未说明的信息必须省略或写入 `limitations`。
- 不输出“值得跑”“推荐报名”等替用户决策的结论。
- 不重复赛事详情已有的大段判断文案。
- 输出严格 JSON。

复用当前 `AI_INGEST_PROVIDER`、模型、Base URL 和密钥配置；将通用 AI 客户端配置提取为共享模块，避免 GLM、DeepSeek、OpenAI 逻辑复制。

### 人工发布

- 生成结果保存为 `draft`。
- 管理员可编辑摘要、关键点和限制说明，不能修改内容哈希、AI 模型和抓取时间。
- 发布时把同赛事原 `published` 版本改为 `superseded`，当前草稿改为 `published`。
- apply 必须携带草稿 `updatedAt`，冲突返回 HTTP 409。
- 发布、编辑、废弃均写管理员操作日志。
- V0.4.5 创建新的 open 变更告警时，把当前 published 摘要设置 `staleAt`；仍可展示旧摘要，但原生来源摘要页顶部显示“来源近期有更新，摘要待复核”。
- 新摘要发布后 `staleAt=null`。

## Interfaces

### 匿名选择与公开数量

`PUT /api/event-choices`

```ts
{
  userKey: string; // 1-100 字符
  eventId: string;
  choice: 'interested' | 'considering' | 'registered';
}
```

- 仅接受当前公开的未来大湾区赛事。
- 使用 upsert，重复设置仍返回成功。
- 响应包含更新后的 `choiceCounts`，便于详情页立即刷新数量。

`GET /api/event-choices?userKey=<key>&choice=<optional>`

- 最多 100 条，按比赛日期升序。
- 赛事不再公开时返回 `event: null, eventAvailable: false`，不泄露归档详情。

`GET /api/event-choices/:eventId?userKey=<key>`

- 只返回当前匿名用户对该赛事的选择；未选择时返回 `{ choice: null }`。
- 该接口失败不影响赛事详情和公开数量展示。

`DELETE /api/event-choices/:eventId?userKey=<key>`

- 幂等返回 `{ removed: true, choiceCounts }`。

现有 `GET /api/events/:id` 增加：

```ts
{
  choiceCounts: {
    interested: number;
    considering: number;
    registered: number;
    total: number;
  }
}
```

- `choiceCounts` 从第 1 票开始返回准确整数，零票返回 `0`，不设置最低展示门槛。
- 聚合查询只使用 `eventId + choice`，不返回参与者记录或用户标识。
- 赛事详情响应保持完全公开，不接收 `userKey`；当前用户选择通过独立选择接口读取，避免缓存混入用户数据。
- `total` 为三个互斥状态之和；同一用户切换状态不增加总人数。
- 详情页固定说明“匿名意向统计，不代表官方报名人数”；`registered` 展示为“已报名（用户自报）”。

### 管理后台来源摘要

`POST /api/admin/events/:id/source-summaries/generate`

- 仅接受赛事已有、已核验的 `sourceUrl/officialUrl`，请求体不能提交任意 URL。
- 抓取和 AI 生成完成后返回新草稿。
- 相同 `contentHash + promptVersion` 已存在时返回现有版本，不重复调用模型。
- `event_operator/content_reviewer/super_admin` 可执行。

`PUT /api/admin/source-summaries/:id`

```ts
{
  summary: string;
  keyPoints: string[];
  limitations?: string | null;
  expectedUpdatedAt: string;
}
```

`POST /api/admin/source-summaries/:id/publish`

```ts
{
  expectedUpdatedAt: string;
  note: string;
}
```

`GET /api/admin/events/:id/source-summaries`

- 返回版本历史、生成依据、模型、抓取时间、发布人和 stale 状态。
- 不返回完整来源正文。

### 公开来源摘要

`GET /api/events/:id/source-summary`

- 仅允许当前公开赛事。
- 只返回 published 版本。
- 无 published 版本返回 HTTP 404。
- 返回来源名称、标题、摘要、关键点、限制说明、来源 URL、basis、抓取时间、生成时间、stale 状态和固定合规提示。
- 来源 URL 只允许 `http/https`；小程序不直接内嵌任意外部网页。

### 小程序公开赛事接口

现有列表和详情增加：

```ts
{
  hasSourceSummary: boolean;
  sourceSummaryStale: boolean;
}
```

不把摘要正文塞进列表或详情，避免重复传输。

`POST /api/interactions` 增加允许 `source_summary_open/source_summary_view/source_original_link_copy`；后台 7/30 天指标增加摘要入口用户、成功加载用户、加载成功率和原始来源链接复制次数。

## User Experience

### 赛事详情

- 在赛事基础信息和“信息有误？反馈纠错”之间增加“查看来源详情”。
- 只有 `hasSourceSummary=true` 时显示入口。
- stale 时入口旁显示“待复核”，不显示具体内部告警。
- 点击进入 `pages/source-summary/index?eventId=<id>`。
- 底部栏继续只保留“前往官方确认、收藏、分享”。

### 原生来源摘要页

页面顺序固定：

1. 赛事名称与“信息来源摘要”。
2. 来源名称、来源标题、抓取时间。
3. 100-400 字摘要。
4. 2-6 条来源关键点。
5. 限制说明或“来源未明确的信息请以前往官方页面为准”。
6. “复制原始来源链接”入口。
7. “AI 整理，仅供参考，报名以官方为准。”。

stale 时在顶部增加提示：“来源近期有更新，本摘要正在复核，请以原始来源为准。”

页面使用原生 WXML、WXSS 和标准滚动容器。复制成功后弹窗说明：“原始来源链接已复制，请在浏览器或微信中打开，并以来源原文为准。”加载失败保留当前页面并提供重新加载和返回赛事详情。

### 匿名选择

- 在判断信息后增加紧凑“我的选择”区域。
- 使用三段控件：想跑、观望、已报名；每项同时显示公开数量，例如“想跑 18”。
- 当前用户的选中状态仅自己可见，三个聚合数量对所有用户可见。
- 控件下方固定显示“匿名意向统计，不代表官方报名人数”；“已报名”补充“用户自报”。
- 再次点击当前状态时确认清除。
- 新增 `pages/choices/index`，支持全部、想跑、观望、已报名筛选。
- “我的下一场”优先级：最近已报名 -> 最近想跑 -> 第一条收藏。
- “我的”页展示三种状态数量和“我的选择”入口。
- 赛事列表本阶段不显示选择数量，避免重复聚合查询和列表信息过密。
