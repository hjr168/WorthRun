# WorthRun V0.4.5 赛事变更监测与发布后复核 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 让已发布赛事在官方来源发生日期、距离、报名状态、截止时间或官方入口变化时进入人工复核闭环，并向用户准确表达信息新鲜度。

**Architecture:** 在现有赛事源抓取和候选持久化链路中增加轻量差异检测。官方或可信来源再次命中已采纳候选时，只更新“来源检查时间”或创建去重的变更告警，不自动修改公开赛事；管理员通过预览和乐观锁确认后才能应用变更。小程序继续复用现有赛事接口，只增加来源检查和复核状态字段。

**Tech Stack:** pnpm monorepo、TypeScript、Express、Prisma/PostgreSQL、React + Ant Design、微信小程序原生框架、Vitest、PM2、Nginx。

---

## Summary

V0.4.4 上线后，线上已有 11 场未来大湾区赛事，近 7 天详情访问 17 次、官方入口复制 9 次，核心决策路径已经可用。当前最大的产品风险是：来源重新抓取已审核候选时会计入 `skippedReviewed`，已发布赛事后续发生日期、报名状态或官方链接变化时，没有独立的运营复核入口。

本阶段优先建立“来源复查 -> 发现差异 -> 人工确认 -> 更新公开赛事 -> 留痕”的闭环，不进入天气、评论、公开投票、订阅消息或官方报名交易。

验收目标：

- 官方或可信来源重新命中已采纳候选时，可以识别关键字段变化。
- 没有变化时仅刷新来源检查时间，不改赛事业务字段和 `updatedAt` 展示口径。
- 有变化时创建可去重告警，不自动覆盖已发布赛事。
- 管理员可以查看差异、证据和来源，并预览后应用、归档或忽略。
- 开放告警存在时，小程序显示“信息复核中”，但仍保留“前往官方确认”。
- 赛事详情显示“来源最近检查”，不把自动抓取描述成人工核验。
- 继续保持单个 API PM2 进程，不新增 cron、队列、Redis 或常驻 worker。
- API RSS 不超过 220MB，单次来源任务峰值不超过 180MB。

## Product Boundaries

- 只有 `official` 和 `trusted` 来源可以触发已发布赛事变更告警。
- `community`、`secondary` 和 `unknown` 来源继续只负责发现候选，不能改变已发布赛事状态。
- 自动检测只比较来源明确提供的字段；空值不覆盖现有非空值，也不产生“字段被删除”告警。
- 日期、距离、报名状态、报名截止时间和官方入口允许在管理员确认后应用。
- 命中取消、延期关键词时只生成高优先级信号；取消必须由管理员确认后归档，延期必须补齐新日期后才能应用。
- 变更告警不直接发送给匿名用户，不新增公开告警列表接口。
- 固定保留“AI 整理，仅供参考，报名以官方为准。”和“前往官方确认”。

新增 `EventChangeAlert`：

```prisma
model EventChangeAlert {
  id                String                 @id @default(cuid())
  eventId           String                 @map("event_id")
  sourceId          String                 @map("source_id")
  sourceRunId       String?                @map("source_run_id")
  sourceCandidateId String?                @map("source_candidate_id")
  status            EventChangeAlertStatus @default(open)
  severity          EventChangeSeverity
  changedFields     String[]               @map("changed_fields")
  beforeValue       Json                   @map("before_value")
  afterValue        Json                   @map("after_value")
  evidence          Json
  sourceUrl         String?                @map("source_url")
  fingerprint       String
  reviewedBy        String?                @map("reviewed_by")
  reviewedAt        DateTime?              @map("reviewed_at")
  reviewNote        String?                @map("review_note")
  createdAt         DateTime               @default(now()) @map("created_at")
  updatedAt         DateTime               @updatedAt @map("updated_at")

  event           Event           @relation(fields: [eventId], references: [id], onDelete: Cascade)
  source          EventSource     @relation(fields: [sourceId], references: [id], onDelete: Cascade)
  sourceRun       EventSourceRun? @relation(fields: [sourceRunId], references: [id], onDelete: SetNull)
  sourceCandidate EventCandidate? @relation(fields: [sourceCandidateId], references: [id], onDelete: SetNull)

  @@unique([eventId, sourceId, fingerprint])
  @@index([status, severity, createdAt])
  @@index([eventId, status])
  @@map("event_change_alerts")
}
```

`EventSource` 和 `EventCandidate` 补充对应关系字段。告警快照只保存可比较字段和最多 10 条证据，不复制完整 HTML、PDF 或原始响应。

## Interfaces

### `GET /api/admin/event-change-alerts`

查询参数：

```ts
{
  page?: number;
  pageSize?: number; // 1-50
  status?: 'open' | 'applied' | 'dismissed' | 'archived_event' | 'superseded';
  severity?: 'normal' | 'important' | 'critical';
  changedField?: 'eventDate' | 'distanceItems' | 'signupStatus' | 'signupDeadline' | 'officialUrl' | 'cancellationSignal' | 'postponementSignal';
  search?: string;
}
```

返回分页告警、关联赛事、来源名称、差异快照、证据和 `updatedAt`。

### `GET /api/admin/event-change-alerts/summary`

返回：

```ts
{
  open: number;
  critical: number;
  important: number;
  stalePublishedEvents: number;
  checkedWithin7Days: number;
  appliedWithin30Days: number;
}
```

`stalePublishedEvents` 定义为：官方或可信来源的未来已发布赛事，`sourceCheckedAt` 为空或早于北京时间 14 天前。

### `POST /api/admin/event-change-alerts/:id/resolve`

dry-run 请求：

```ts
{
  dryRun: true;
  action: 'apply_fields' | 'dismiss' | 'archive_event';
  fields?: Array<'eventDate' | 'distanceItems' | 'signupStatus' | 'signupDeadline' | 'officialUrl'>;
  note: string;
}
```

apply 请求额外携带：

```ts
{
  expected: {
    alertUpdatedAt: string;
    eventUpdatedAt: string;
  };
}
```

规则：

- `apply_fields` 只允许选择告警实际变化且来源提供非空值的字段。
- 应用后的赛事仍须满足未来大湾区和发布规则；不满足时返回 HTTP 400。
- `archive_event` 仅允许 critical 取消信号，由 `super_admin` 或 `event_operator` 执行。
- `dismiss` 必须填写 4-500 字备注。
- 快照冲突返回 HTTP 409，不修改告警和赛事。
- 每次处理写入 `AdminOperationLog`，包含告警、字段、前后值和备注。

### 公开赛事接口

现有 `/api/events` 和 `/api/events/:id` 结构保持兼容，增加：

```ts
{
  sourceCheckedAt: string | null;
  sourceReviewPending: boolean;
}
```

公开接口不返回告警差异、来源原文、管理员备注或内部风险类型。

## Test Matrix

- 差异规范化：日期、距离顺序、重复距离、URL 跟踪参数、时区和空值。
- 可信边界：official/trusted 产生告警，community/secondary/unknown 不产生告警。
- 已审核保护：accepted、merged 和 rejected 候选不会被重新激活或覆盖。
- 告警去重：相同 fingerprint 重复抓取不增加开放告警。
- 告警升级：新 fingerprint 生成新告警，旧告警按处理结果保留追溯。
- 无变化检查：只更新 `sourceCheckedAt`，不改变赛事展示字段。
- 人工处理：dry-run、字段白名单、部分字段应用、忽略、归档、权限和操作日志。
- 并发保护：告警或赛事在预览后变化时返回 HTTP 409。
- 发布边界：过期日期、区域外城市、无官方入口和空距离不能通过应用保持公开。
- 小程序：无检查时间、正常检查、信息复核中、长赛事名和安全区布局。
- 回归：候选归并、批量采纳、批量发布、反馈治理和公开赛事数量不受影响。

## Rollback

- 应用 migration 前保留数据库备份。
- 代码回滚时保留新增告警表和字段，不执行破坏性 down migration。
- 若差异检测产生噪声，可通过配置关闭“已审核候选变更检测”，原抓取和候选流程继续运行。
- 若后台页面异常，可隐藏导航入口；告警数据仍保留，公开赛事不受影响。
- 若小程序字段尚未发布，新增公开接口字段保持向后兼容。

## Assumptions

- V0.4.5 是数据可信度迭代，不是用户增长功能版本。
- 当前 11 场未来大湾区赛事作为公开接口回归基线，除真实过期或管理员确认归档外不应减少。
- 当前 7 条待处理候选继续走现有审核流程，不因本计划自动采纳。
- 只有来源明确提供的值参与比较，AI 推断出的空值或不确定值不覆盖人工核验结果。
- 本阶段不开发微信订阅消息、天气、赛事对比、公开投票、评论、积分或报名闭环。
- 不覆盖或提交现有 `apps/miniapp/project.private.config.json` 用户修改。
