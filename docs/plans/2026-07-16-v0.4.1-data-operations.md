# WorthRun V0.4.1 数据运营闭环 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 将公开赛事与自动抓取统一收口到北京时间未来的大湾区赛事，并提供可追溯数据治理和最小行为统计。

**Architecture:** 共享包提供地域和日期规则，API 在抓取、采纳、发布和公开查询边界复用规则。PostgreSQL 保存过滤计数和匿名日级行为，后台提供治理预览、数量保护应用及 7/30 天运营指标。

**Tech Stack:** TypeScript、Express、Prisma/PostgreSQL、React/Ant Design、微信小程序、Vitest、系统 cron、PM2。

---

### Task 1: 数据边界与抓取过滤

- 在 `packages/shared/src/index.ts` 增加大湾区 9+2 城市别名和北京时间日期函数。
- 在候选持久化、候选采纳、赛事发布和公开查询复用统一规则。
- 为来源运行增加过期与区域外过滤计数，补 migration 和测试。

### Task 2: 可追溯数据治理

- 实现 dry-run 默认的数据治理服务和 CLI。
- apply 必须携带预览数量；事务内复算，不一致返回冲突。
- 只执行驳回或归档并写操作日志，不物理删除。

### Task 3: 最小行为统计

- 新增匿名日级 `EventInteraction`，仅记录详情访问与官方入口复制。
- 小程序静默上报，服务端 upsert 去重。
- 后台展示 7/30 天详情、官方入口、收藏、分享和偏好指标。

### Task 4: 验证与部署

- 运行 `pnpm test`、`pnpm typecheck`、API/admin/miniapp 构建。
- ECS 先备份、迁移、部署，再 dry-run 和 apply 数据治理。
- 验证公网 API、后台生产资产、cron RSS 和 PM2 健康状态。
