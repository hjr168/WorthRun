# WorthRun V0.5.3 用户体系、增长基线与赛事提醒

## 交付范围

- 小程序通过 `wx.login` 静默绑定微信用户，失败时保留匿名浏览和旧版 `userKey` 兼容。
- OpenID 使用 AES-256-GCM 密文和 HMAC 哈希保存，不保存 `session_key`。
- 首次绑定幂等合并偏好、收藏、赛事选择、反馈和分享数据。
- 新增原生小程序个人资料、我的提醒页，以及赛事详情提醒入口。
- 头像使用 UniCloud 支付宝云 URL 化函数直传，ECS 只签发一次性凭证和接收签名回调。
- 后台新增“用户管理”和“增长与提醒”，仅 `super_admin` 可见。
- 增长基线从 V0.5.3 开始记录日活、新用户、D1/D7、行为漏斗和分享归因，不回填历史留存。
- 报名与赛前提醒通过一次性任务发送，不新增 PM2 常驻进程。

## 分阶段开启

1. 备份 PostgreSQL，部署迁移和 API，保持 `USER_SYSTEM_ENABLED=false` 与 `REMINDER_FEATURE_ENABLED=false`。
2. 部署 UniCloud 函数，配置三类小程序合法域名，更新隐私政策和微信隐私保护指引。
3. 生成用户 token、OpenID HMAC 和 AES 密钥，通过体验版开启用户体系。
4. 真机验证静默注册、跨设备恢复、头像直传、禁用与注销。
5. 申请并配置两个订阅消息模板，真机验收通过后再开启提醒功能和 cron。

## 资源边界

- API 保持单个 PM2 进程，目标 RSS 小于 220MB。
- `reminder:send-due` 使用 96MB Node heap，每批最多 30 条。
- 头像文件不经 ECS；主 API 只处理凭证、fileID 和短期 URL。
- 过期头像凭证和分享 token 由现有 `feedback:maintenance` 清理，不增加新的维护 cron。
