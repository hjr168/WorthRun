# 真实数据内测流程

本流程用于体验版前的真实赛事数据内测。V0.1 仍由人工维护赛事信息，不接入 AI 自动采集、自动解析或自动发布。

## 1. 准备真实数据 CSV

1. 复制待填写模板：

   ```bash
   cp docs/real-events-template.csv docs/real-events.local.csv
   ```

2. 人工填写至少 10 条真实赛事。
3. 每条赛事都要核验赛事日期、距离项目、官方入口、来源名称和来源链接。
4. `officialUrl` 不得使用 `example.com`、搜索结果页、截图或无法追溯的转述链接。

## 2. dry-run 校验

先只校验，不写入数据库：

```bash
pnpm db:import-events -- ./docs/real-events.local.csv --dry-run
```

检查输出中的将新增、将更新和失败行。失败行必须回到 CSV 修正后再次 dry-run。

## 3. 正式导入

dry-run 无失败后再正式导入：

```bash
pnpm db:import-events -- ./docs/real-events.local.csv
```

导入只创建或更新后台草稿赛事，不会自动发布。

## 4. 后台人工核验

1. 打开后台赛事库。
2. 逐条核验赛事名称、城市、日期、距离、报名状态、官方入口、来源名称、来源等级。
3. 检查跑前判断和至少 1 条判断理由。
4. 检查报名前确认清单是否完整。
5. 检查合规提示仍为：`AI 整理，仅供参考，报名以官方为准。`
6. 检查官方入口文案仍为：`前往官方确认`。
7. 处理发布前检查提示中的待补充或风险词项。

## 5. 发布内测赛事

至少发布 5 条已核验赛事。发布前确认没有使用 `example.com`，也没有出现报名直达文案。

## 6. 小程序联调

1. 将小程序 API 配置切换到测试 API。
2. 在微信开发者工具中重新编译。
3. 首页、赛事列表、赛事详情、收藏、反馈、我的页面逐项检查。
4. 真机预览测试官方入口复制弹窗、收藏、反馈提交和错误态重新加载。
5. 按 `docs/MINIAPP_INTERNAL_TEST_CHECKLIST.md` 逐项验收。

## 7. 问题记录与修复

记录每个问题的页面、赛事、复现步骤、预期结果和实际结果。修复后重新 dry-run、导入、后台核验和真机测试，再准备体验版。
